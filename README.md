## 모델링

### 상품

- `Product`는 식별자와 `DisplayedName`, 가격, 화폐단위를 가진다.
- `DisplayedName`에는 `Profanity`가 포함될 수 없다.

### 메뉴

- `MenuGroup`은 식별자와 이름을 가진다.
- `Menu`는 식별자와 `Displayed Name`, 가격, `MenuProducts`를 가진다.
- `MenuProducts`는 수량, `Product`의 식별자와 `Product`의 가격을 가진다.
- `Menu`는 특정 `MenuGroup`에 속한다.
- `Menu`의 가격은 `MenuProducts`의 금액의 합보다 적거나 같아야 한다.
- `Menu`의 가격이 `MenuProducts`의 금액의 합보다 크면 `NotDisplayedMenu`가 된다.
- `MenuProduct`는 가격과 수량을 가진다.

### 주문 마스터

- `OrderMaster`는 식별자와 `OrderStatus`, 주문 시간, `OrderType`,  `OrderLineItems`를 가진다.
- `OrderType`은 매장 주문, 배달 주문, 포장 주문으로 나뉜다.
- `OrderType`가 `배달 주문`인 경우 접수 대기 ➜ 접수 ➜ 서빙 ➜ 배달 ➜ 배달 완료 ➜ 계산 완료 순서로 진행된다.
- `OrderType`가 `매장 주문`, `포장 주문`인 경우 접수 대기 ➜ 접수 ➜ 서빙 ➜ 계산 완료 순서로 진행된다.
- `OrderMaster`는 접수 대기 ➜ 접수 ➜ 서빙 ➜ 계산 완료 순서로 진행된다.
- `OrderLineItem`는 가격과 수량을 가진다.
- `OrderLineItem`의 수량은 기존 `Order`를 취소하거나 변경해도 수정되지 않기 때문에 0보다 적을 수 있다.
- 메뉴가 노출되고 있으며 판매되는 메뉴 가격과 일치하면 `Order`가 생성된다.
- `OrderLineItem`는 `MenuId', `메뉴가격`, `수량` 가격과 수량을 가진다.
- `OrderLineItem`의 매장주문의 수량은 0보다 작을 수 있다.
- `OrderLineItem`의 배달 포장 주문의 수량은 0보다 커야 한다.

### 매장 주문

- `OrderTable`은 식별자와 이름, `NumberOfGuests`를 가진다.
- `OrderTable`에는 손님이 앉을 수 있다.
- `OrderTable`의 추가 `Order`는 `OrderTable`에 계속 쌓이며 모든 `Order`가 완료되면 `EmptyTable`이 된다.
- `EmptyTable`인 경우 `NumberOfGuests`는 0이며 변경할 수 없다.
- `EatInOrder`은 식별자, `OrderMasterId`, `OrderTableId`를 가진다.
- `OrderMaster` 정보가 없으면 `EatInOrder` 를 등록 할 수 없다.
- `OrderTable`이 등록되 있지 않으면 매장 주문을 할 수 없다.
- `OrderTable`에 손님이 앉아있지 않으면 매장 주문을 할 수 없다.
- `OrderMaster` 정보가 없으면 `EatInOrder` 에서 접수를 할 수 없다.

### 배달 주문

- `DeliveryOrder`는 식별자와 `orderMasterId`, 배달 주소 를 가진다.
- `OrderMaster` 정보가 없으면 `DeliveryOrder` 를 등록 할 수 없다.
- `DeliveryOrder`는 `OrderMaster`가 등록 후에 등록 된다.
- `DeliveryOrder`가 접수되면 `DeliveryAgency`가 호출된다.

### 포장 주문

- `TakeOutOrder`는 식별자와 `orderMasterId`, 배달 주소 를 가진다.
- `OrderMaster` 정보가 없으면 `TakeOutOrder` 를 등록 할 수 없다.
- `TakeOutOrder`가 접수되면 `DeliveryAgency`가 호출된다.
